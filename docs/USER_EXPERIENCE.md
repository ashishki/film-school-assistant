# Film School Assistant — User Experience Principles

## 1. UX Goal

The assistant should make creative work feel less chaotic, more continuous, and more workable.

The target feeling is:
- clear
- steady
- non-chaotic
- supportive without flattery
- useful in motion, not just useful in retrospect

## 2. Emotional Layer

This product is not only about functional task management.

For the target user, fragmented notes and deadlines create emotional drag:
- loss of continuity
- low-grade overwhelm
- friction restarting work
- guilt around scattered unfinished thinking

So the assistant should reduce cognitive noise, not add more of it.

## 3. Tone of the Assistant

The assistant should feel:
- calm
- precise
- grounded
- creatively literate
- concise

It should not feel:
- chirpy
- overly motivational
- generic productivity-coach-like
- mystical or pseudo-deep
- robotic and bureaucratic

## 4. Continuity Principle

The assistant should help the user feel:
- "I know where this project stands."
- "I know what changed."
- "I know what matters next."

That is more important than adding more commands.

## 5. Progress Principle

The assistant should make progress visible.

Examples:
- confirm what was captured and where it lives
- show project context when relevant
- make weekly summaries directional, not just descriptive
- make next-step suggestions feel grounded in actual stored work

## 6. Anti-Goals

Avoid these UX outcomes:
- a wall of raw data
- command-centric experience with no continuity framing
- generic AI compliments
- excessive verbosity
- false certainty about user intent
- fake "creative partnership" language unsupported by the system

## 7. Why Functional-Only UX Is Not Enough

A purely functional workflow can store the work and still fail the user.

If the user still feels scattered after using the system, then the product is underperforming even if the CRUD flows work.

This is a creative assistant. Clarity, continuity, and reduced chaos are part of the product, not decorative extras.

## 8. Design Implications for the Next Phase

Phase 1 should improve:
- confirmation wording
- project-awareness in replies
- weekly digest structure
- tone consistency
- orientation cues after capture, review, and edit actions

Phase 1 should not default to:
- new surfaces
- new storage systems
- generic "AI features"
