# Film School Assistant — Product Overview

## 1. What This Product Really Is

Film School Assistant is a private AI assistant for directors and serious creative thinkers who need help turning scattered inputs into structured, continuous progress.

It is a workflow product, not a chat novelty product.

Its current form is Telegram-first because capture speed matters more than interface breadth at this stage.

## 2. Target User

Primary user now:
- one director in training
- one film student
- one solo creative thinker juggling multiple active projects

The product is intentionally optimized for a single operator-user, not for a team.

## 3. Job To Be Done

When I am juggling ideas, deadlines, coursework, and half-formed project thinking,
help me capture fast, keep context attached to the right project, recover what matters later,
and reduce the feeling that my work is dissolving into fragments.

## 4. Core Outcomes

The product should create four outcomes:

1. **Capture without friction**
   The user should be able to send a thought before it evaporates.

2. **Structure without administrative drag**
   The system should turn messy inputs into usable project-linked records.

3. **Continuity across sessions**
   The user should be able to come back after a gap and quickly regain orientation.

4. **Forward motion**
   The system should not just store work; it should help surface the next meaningful step.

## 5. Product Boundaries

### What it is

- a private creative workflow assistant
- a continuity aid for directing and film-school work
- a bounded AI system with deterministic operational scaffolding

### What it is not

- a general productivity app
- a team collaboration suite
- a SaaS platform
- a pure note-taking bot
- a general-purpose autonomous creative agent

## 6. Differentiation

The differentiation is not "it uses AI."

The differentiation is:
- fast capture in a low-friction interface
- project-scoped creative structure
- local voice privacy
- bounded AI reflection instead of vague AI chat
- continuity-oriented workflow design

## 7. Why Telegram Is Still Correct

Telegram is correct now because the main high-frequency use case is capture:
- text notes
- voice notes
- fast save and confirm flows
- reminders back into the same channel

Telegram is not the long-term identity of the product. It is the current best surface for minimum-sufficient delivery.

## 8. Product Boundaries for the Next Year

### In

- better creative continuity
- stronger memory artifacts
- better guidance and reflection
- stronger product legibility

### Out

- multi-user expansion
- team workflow
- marketplace integrations
- speculative "director SaaS"
- agentic autonomy beyond current bounded behavior

## 9. Honest Current Strengths

- the implementation foundation is real
- the product already has structured entities and recurring workflows
- local voice handling is a real privacy and packaging strength
- the system is narrow enough to remain coherent

## 10. Honest Current Weaknesses

- the product story is weaker than the implementation
- continuity support is still shallower than the positioning aspires to be
- the experience can still feel like stored fragments plus commands
- documentation has been stronger on control than on product meaning
