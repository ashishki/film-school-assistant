## Eval Dimension 1 — Project-Context Question

**Scenario**
User has active project "Короткометражка «Стекло»" with 3 notes and 1 idea stored.
User runs `/memory` → memory generated and cached.
User then asks in chat: "что у меня сейчас по стеклу?"

**Without memory (Phase 1 baseline)**
`handle_chat` receives base system prompt only.
The LLM has no project context and must rely entirely on tool calls to list notes/ideas.
It may call `list_notes` and return a raw list — functional but no continuity framing.
The LLM does not know where the project stands overall.

**With memory (Phase 2)**
`handle_chat` reads `project_memory` row for the active project.
`_build_system_prompt` prepends: `Контекст проекта: {summary_text}`.
The LLM receives the project state before the tool call loop.
It can anchor its response to the project's current state and use tools
to supplement rather than reconstruct context from scratch.

**Verdict: IMPROVED**

The LLM now enters the chat with grounded context rather than a blank slate.
For a project-state question, the first response should be materially more oriented.

**Hallucination risk**
The injected text is `summary_text` generated by `/memory` from stored records only.
The system prompt instructs: "Не выдумывай сущности, которых ты не получал через инструменты."
The memory generation prompt instructs: "Не выдумывай ничего, чего нет в данных."
Two independent constraints. Risk is low if memory was generated honestly.

**Residual gap**
Memory is per-request read from DB but not per-session refreshed.
If the user adds a note during the chat session, the memory in the system prompt
is stale for the remainder of that session. Acceptable — user can `/memory` to refresh.

## Eval Dimension 2 — Capture Confirmation

**Scenario**
User sends a free-text capture: "идея — снять сцену в тумане, ранее утро"
NL handler extracts entity_type=idea, project resolved to active project.
User confirms → `_confirm_success_text` runs.

**Without memory (Phase 1 baseline)**
Confirmation: "Идея сохранена → Короткометражка «Стекло»"
No memory involvement — correct, this is a deterministic path.

**With memory (Phase 2)**
Confirmation path is unchanged. Memory is not injected into confirmation flows —
only into `handle_chat`. This is correct behavior per MR-3.

**Verdict: UNCHANGED — by design**

Memory injection is scoped to chat context only. Capture confirmation remains
fully deterministic and is not affected by Phase 2. No regression.

## Eval Dimension 3 — Idea Review

**Scenario**
User runs `/review 7` (idea id=7 belongs to active project).
`review_idea` calls LLM with `REVIEW_SYSTEM_PROMPT`.

**Without memory**
Review uses a fixed system prompt with no project context.
The LLM critiques the idea in isolation.

**With memory (Phase 2)**
`reviewer.py` does not use `handle_chat` — it calls `complete_json` directly.
Memory injection is not applied to the review path.
This is correct: the review prompt is already project-specific via idea content,
and injecting project memory into a structured critique prompt risks polluting the
fixed output schema.

**Verdict: UNCHANGED — by design**

Review path correctly excluded from memory injection scope.
Phase 3 could consider injecting project context into review if evidence supports it,
but it is not a Phase 2 gap.

## Hallucination / Over-injection Findings

| Finding | Severity | Notes |
|---------|----------|-------|
| Memory text is truncated only by the 200-word LLM output limit, not by a hard character cap | Low | System prompt says max 200 words. In practice, model may occasionally exceed this. A hard slice in `_build_system_prompt` could be added as a guard in Phase 3. |
| No staleness indicator shown to the user in chat | Low | The user does not know if the injected memory is fresh or stale. `/memory` always shows `_(актуально)_` when cache hit — sufficient for now. |
| `item_count_snapshot` uses notes+ideas+active-deadlines only; archived deadlines or reviews don't increment it | Low | If user runs `/review` on an idea, memory staleness is not detected. Acceptable — review history is a different signal. |

No critical or high-severity hallucination findings.
