## 1. System Identity

**Film School Assistant** — single-user Telegram bot for structured creative workflow management. Designed for a film school student. Privacy-first (local STT, no external audio transmission).

---

## 2. Current Functionality (Implemented)

### 2.1 Explicit Commands

| Command | Syntax | Behavior |
|---|---|---|
| `/note <text>` | Free text after command | Creates note, tags to active project if set |
| `/idea <text>` | Free text after command | Creates idea with review_status='unreviewed' |
| `/deadline <title> due <date>` | Due date parsed (ISO, "next Friday", "20 апреля") | Creates deadline with status='active' |
| `/homework <title> due <date> [course:<name>]` | Regex-parsed | Creates homework with status='pending' |
| `/project <name>` | Fuzzy match on project name/slug | Sets active project context in session state |
| `/projects` | No args | Lists all projects |
| `/list notes\|ideas\|deadlines\|homework [project:<name>]` | Optional project filter | Returns up to 20 items per type |
| `/review <idea_id>` | Integer ID | Sends idea to Sonnet LLM, returns structured critique, stores review_history |
| `/done_deadline_<id>` | ID embedded in command | Sets deadline status → 'done' |
| `/dismiss_deadline_<id>` | ID embedded in command | Sets deadline status → 'dismissed' |
| `/confirm` | No args | Saves pending entity (from voice or NL flow) |
| `/edit <field> <value>` | Field name + value | Edits pending entity before confirmation |
| `/discard` | No args | Clears pending entity without saving |
| `/help` | No args | Shows command list |

### 2.2 Voice Input

- Accepts Telegram voice messages (OGG format)
- Downloads and converts to WAV (16kHz mono via FFmpeg)
- Transcribes locally via Whisper small model
- Extracts entity_type/content/project_hint/due_date via Haiku LLM
- Presents transcript + structured extraction to user for confirmation
- Stores audit trail: voice_inputs → transcripts → parsed_events

### 2.3 Free-Text NL Input

- Accepts non-command text messages when no pending entity exists
- Routes via Haiku LLM extraction (same as voice)
- If entity_type ambiguous: shows inline type-selection keyboard
- Confirmation flow identical to voice

### 2.4 Inline Buttons

- `[Confirm]` / `[Discard]` — confirm or reject pending entity
- `[type_note]` / `[type_idea]` / `[type_deadline]` / `[type_homework]` — select entity type when ambiguous

### 2.5 Natural Language Confirm/Discard Words

- Russian: "да", "ок", "конечно", "сохрани", "нет", "стоп", "отмена", "удали"
- English: "yes", "ok", "sure", "save", "no", "stop", "cancel", "delete"
- Detected in group=1 handler before NL routing

### 2.6 Automated Scheduled Behaviors

| Behavior | Schedule | Description |
|---|---|---|
| Deadline reminders | Daily 08:00 | Sends Telegram message for deadlines at 7d/3d/1d/0d before due. Idempotent via reminder_log. |
| Weekly summary | Monday 09:00 | 7-day activity snapshot + upcoming deadlines + stalled projects → Sonnet narrative → sent to user |
| Audio cleanup | Daily 03:00 | Deletes voice files older than AUDIO_RETENTION_DAYS (default 30 days) |
| DB backup | Daily 02:00 | SQLite .backup + integrity check, 7 copies retained |

### 2.7 Idea Review Output Format

```
CORE IDEA: ...
DRAMATIC CENTER: ...
WEAK POINTS: ...
QUESTIONS:
  1. ...
  2. ...
  3. ...
NEXT STEP: ...
```

### 2.8 Security Guard

- All incoming Telegram updates checked against TELEGRAM_ALLOWED_CHAT_ID
- Non-matching chat_id: silently dropped (no response)
- Handler group=-1000 ensures guard runs before any handler

## 3. Expected Behavior (Inferred from Code)

- `/list` returns maximum 20 items (hardcoded limit in db.py)
- Pending entity blocks new voice input — bot replies with rejection message until resolved
- Active project context persists for the session in memory; lost on bot restart
- Deadline reminders use days_before deduplication: once per (deadline_id, days_before) pair
- Review stores full prompt + response JSON for audit; does not re-use cached response
- Summary is skipped if weekly_reports record for current week_start already has sent_at set

## 4. Missing Features

| Feature | Priority | Notes |
|---|---|---|
| `/new_project <name>` command | High | Projects currently must be seeded via SQL. Blocks core workflow for new users. |
| Editing saved entities | High | Once confirmed, entities are immutable in the bot. User must discard + recreate. |
| `/list` filtering by status | Medium | e.g., `/list homework status:pending`. Currently shows all. |
| Pagination for `/list` | Medium | Hard limit of 20; no offset or "next page" support. |
| Search within entity type | Medium | No keyword search in notes/ideas. Only project filter. |
| Project deletion / archival | Medium | Projects are permanent. No soft-delete. |
| Configurable reminder buckets | Low | REMINDER_BUCKETS=(7,3,1,0) hardcoded in send_reminders.py |
| Multi-entity voice extraction | Low | Voice containing two entities (e.g., deadline + note) creates only one. |
| Tags / labels on entities | Low | No tagging system. Single project association only. |
| Rate limiting on LLM calls | Low | No backpressure. Cost guardrail absent. |

## 5. Acceptance Criteria

### Core Commands
- `/note text` → note created, reply includes note ID and active project name (if set)
- `/idea text` → idea created with review_status='unreviewed'
- `/deadline title due date` → deadline created, due_date parsed correctly for ISO, Russian, relative English formats
- `/homework title due date [course:X]` → homework created with correct course tag
- `/project name` → fuzzy match resolves to closest project, state updated, confirmation message
- `/review id` → Sonnet review returned in structured format, review_history record created

### Voice Flow
- Voice message → transcript shown to user within reasonable latency (~3-5s on CPU)
- Extracted fields shown in confirmation message
- `/confirm` → entity saved, parsed_events.confirmed=1, voice_input.processed_at set
- `/discard` → pending cleared, no entity saved
- Second voice message while pending → rejected with clear error message

### Scheduled Tasks
- Reminder not sent twice for same (deadline_id, days_before) pair
- Summary not sent if already sent this week
- Audio files older than retention limit deleted; newer files preserved
- Backup created daily; only 7 most recent retained

### Security
- Message from non-allowed chat_id → silently dropped
- Bot does not respond to unauthorized chats under any condition

## 6. Capability-Specific Requirements

### Tool-Use (ON)
- Anthropic API: JSON response required for all extraction calls; non-JSON must trigger retry or fallback
- Whisper: model cached in memory after first load; not reloaded per request
- FFmpeg: temp WAV file deleted after transcription regardless of success/failure
- Telegram API: exponential backoff on send failures (currently MISSING in scripts — see tasks.md)

### RAG (OFF)
- No vector search, no embedding pipeline, no retrieval over corpus. Direct SQLite queries only.

### Agentic (OFF)
- No autonomous loops. All entity creation requires explicit user confirmation (voice/NL) or deterministic command execution.

### Planning (OFF)
- No multi-step planning inside the bot. Weekly summary narrative is text synthesis, not planning.
